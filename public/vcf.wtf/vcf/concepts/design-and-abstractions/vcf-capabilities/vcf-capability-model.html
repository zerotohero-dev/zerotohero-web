VCF Capability Model

Build

  • Deploy a new VCF platform
  • Converge existing virtual infrastructure to a new VCF platform.
  • Deploy a Workload Domain
  • Import an existing vCenter instance to a Workload Domain.
  • Add a vSphere Cluster or an ESX host to a Workload Domain.
  • Configure vSAN Stretched Cluster
  • Manage network connectivity using NSX Edge Clusters and Transit Gateways in vCenter
  • Manage virtual infrastructure in vCenter

Manage (VCA Automation)

  • Licensing
  • Configure IAM and SSO
  • Link vCenter instances for centralized inventory management
  • Manage certificates across the VCF Fleet
  • Manage password across the VCF Fleet
  • Manage tags across the VCF Fleet
  • Manage the configuration settings of vCenter instances and vSphere Clusters.
  • Backup and restore the VCF Fleet and its managed VCF Instances
  • Migrate workloads across Workload Domains and vCenter instances

Operate (VCF Operations)

  • Deploy and configure vSphere Supervisor for running Kubernetes Workloads and VM Workloads.
  • Licensing
  • Configure alerts
  • Perform log analysis and troubleshooting
  • Monitor network operations
  • Monitor storage operations
  • Perform infrastructure diagnostics and VCF health monitoring
  • Configure and use monitoring dashboard and monitoring reports
  • Workload monitoring and observability
  • Manage cost and capacity
  • Mitigate security risks
  • Enforce security compliance
  • Manage infrastructure, identity, and access across consumer organizations

Network and Storage

  • Manage vSAN storage
  • Manage networking in NSX

Platform Operations (VCF Operations)

  • Link vCenter instances for centralized inventory management
  • Manage certificates across the VCF Fleet
  • Manage passwords across the VCF Fleet
  • Manage tags across the VCF Fleet
  • Manage the configuration settings of vCenter instances and vSphere Clusters.

Organization Administration

  • Prepare the collection of VM-base workload Blueprints and infrastructure services for the organization and end users

DevOps

  • Provision VMs
  • Provision Kubernetes clusters
  • Provision vSphere Pods
  • Provision persistent volumes
  • Provision secret stores
  • Install Harbor container registries
