VLAN

A VLAN (Virtual Local Area Network) is a logical segmentation of a Layer-2 network that allows multiple independent broadcast domains to coexist on the same physical switching infrastructure.

A VLAN lets you pretend one physical network is many separate networks.

Devices in different VLANs behave as if they are on different switches, even when plugged into the same one.

How About VXLAN

VCF uses both VLAN and VXLAN, whilee VXLAN (via NSX) is the primary and modern networking model.

  • VLANs → for the underlay (physical network connectivity)
  • VXLAN → for the overlay (tenant / workload networking)

Workloads never live directly on VLANs. They live on VXLAN-backed logical networks provided by NSX.

Underlay (Physical Network)

This is the real physical fabric: Switches, routers, NICs.

  • Uses traditional VLANs
  • Carries:
    • ESX management traffic
    • vMotion
    • vSAN
    • NSX transport traffic (VTEP traffic)
  • Designed to be simple, scalable, and stable

Overlay (Logical Network)

This is where actual tenant and workload networking happens.

  • Implemented using VXLAN
  • Managed by NSX
  • Provides:
    • Logical switches
    • Distributed routing
    • Microsegmentation
    • NAT, LB, firewalling, etc.

L2 Over L3

VXLAN is used to:

  • Create L2 networks over L3 infrastructure
  • Allow massive scale (up to ~16 million segments vs 4096 VLANs)
  • Decouple network topology from physical constraints

In VCF:

  • Every workload network = VXLAN-backed logical segment
  • Encapsulated inside UDP packets
  • Carried across the underlay VLANs

So effectively:

VM / Pod
  ↓
Logical Switch (VXLAN)
  ↓
NSX Transport Node
  ↓
VLAN-backed physical network

Here is a comparison table:

Aspect VLAN VXLAN
Layer L2 L2 over L3
Scale ~4K networks ~16M networks
Used in VCF for Underlay Overlay (tenant networks)
Managed by Physical switches NSX
Tenant isolation Limited Strong
Required for NSX Yes (underlay) Yes (overlay)

Why Do We Need VLANs

Without VLANs:

  • Every device hears every broadcast
  • Security boundaries are weak
  • Networks don’t scale cleanly

With VLANs:

  • Broadcast traffic is contained
  • Security and isolation improve
  • Networks become modular and scalable

How Do VLANs Work

1. VLAN IDs

  • Each VLAN is identified by a VLAN ID (typically 1–4094)
  • VLAN ID = logical network identity

2️. Access Ports vs Trunk Ports

Access Port

  • Carries traffic for one VLAN
  • Frames are untagged
  • Typical for end devices (servers, VMs, appliances)

Trunk Port

  • Carries traffic for multiple VLANs
  • Frames are tagged with VLAN IDs
  • Used between switches, hypervisors, routers

3️. VLAN Tagging (802.1Q)

On trunk links, Ethernet frames include:

  • VLAN ID
  • Priority bits (QoS)

This is how a switch knows which logical network a frame belongs to.

VLAN vs Subnet

One VLAN usually maps to one IP subnet — but they are not the same thing.

Concept VLAN Subnet
OSI Layer Layer 2 Layer 3
Purpose Broadcast isolation IP routing boundary
Addressing MAC-based IP-based
Routing required between VLANs Subnets

VLANs and Routing

  • Devices in the same VLAN can communicate directly
  • Devices in different VLANs require a router or L3 switch
  • This is called inter-VLAN routing

VLAN as an Underlay Network

In VMware-based environments:

  • VLANs are used for underlay networking
  • Examples:
    • Management
    • vMotion
    • Storage (vSAN)
    • Overlay transport (NSX TEPs)

NSX overlays (segments) may sit on top of VLAN-backed transport networks.

So, VLANs are still foundational, even in overlay-driven designs.

Summary

  • VLAN = logical Layer-2 network
  • Provides broadcast isolation
  • Uses tagging on trunk links
  • Distinct from IP subnets
  • Still critical in modern virtualized and cloud platforms
