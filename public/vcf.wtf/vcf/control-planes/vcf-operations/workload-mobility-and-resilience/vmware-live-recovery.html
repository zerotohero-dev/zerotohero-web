VMware Live Recovery

VMware Live Recovery is VMware’s modern, unified disaster-recovery (DR) and cyber-recovery capability, designed to protect VCF-managed workloads against outages, site failures, and ransomware, with minimal downtime and minimal data loss.

VMware Live Recovery = continuous protection + fast failover + clean recovery, natively aligned with VCF operations.

Where Does It Sit in the VCF Stack

VMware Live Recovery is not a hypervisor feature and not just storage replication.

It lives at the platform services / operations layer of VCF.

Layer Role
ESX / vSAN Provides replication primitives
vCenter Inventory, orchestration hooks
VCF Operations Lifecycle, monitoring, automation
VMware Live Recovery DR orchestration, ransomware recovery, failover

What Problems Does VLR Solve

1. Traditional disaster recovery

  • Site failure
  • Cluster outage
  • Maintenance windows gone wrong

2. Ransomware & cyber incidents

  • Encrypted VMs
  • Malicious data corruption
  • Need for clean rollback instead of “latest replica”

Core Capabilities

1. Continuous data protection (CDP)

  • Near-zero RPO (seconds)
  • Change-block tracking
  • Continuous replication rather than periodic snapshots

2. Fast Orchestration and Failover

  • Automated failover / failback
  • Application-aware recovery ordering
  • Network remapping and IP preservation

3. Clean-Point Recovery (anti-ransomware)

  • Roll back to a known-good state
  • Avoid restoring infected replicas
  • Immutable recovery points (depending on the backing storage)

4. Cloud-to-cloud and on-prem ↔ cloud

  • Protect workloads:
    • on-prem → on-prem
    • on-prem → cloud
    • cloud → cloud
  • Strong alignment with hybrid VCF deployments

What Does VLR Protect

Primarily:

  • VM workloads
  • vSAN-backed storage
  • VCF-managed domains

It is not:

  • A Kubernetes-native backup system
  • A replacement for application-level backups
  • A secrets-or-identity-recovery tool

^ all these remain separate concerns.

Workloads Must Recover

You can think of VCF as enforcing invariants:

“Workloads must be recoverable within defined SLOs.”

VMware Live Recovery is the enforcement mechanism for the resilience invariant:

  • If a site fails → recover
  • If data is corrupted → rewind safely
  • If ransomware hits → recover clean, fast, and confidently

What VLR Is Not

VLR is

  • Not vMotion
  • Not VCF Operations HCX
  • Not a backup product
  • Not a Kubernetes-native DR

Those solve mobility or data protection, not platform-level resilience orchestration.

Summary

  • VMware Live Recovery is VCF’s unified DR + cyber-recovery platform
  • Designed for low RPO, fast RTO, and clean recovery
  • Successor mindset to SRM, built for ransomware-era threats
  • Operates at the VCF Operations layer, not the hypervisor
  • Focused on VM workloads, not containers or apps directly
