Gated SDK

VCF Gated SDK is a RESTful SDK around everything VCF.

The VCF Gated SDK refers to the VMware Cloud Foundation Software Development Kit that is not publicly downloadable without authentication: You need to be an authorized VMware/Broadcom customer or partner to access it.

So, “Gated” here means governed access, not technical limitation.

In practice, “gated” means:

  • You must log in to the Broadcom Developer Portal with valid credentials.
  • Access is typically tied to an active support contract or partner agreement.
  • The SDK download link is hidden until your account is verified for entitlement.

VCF Gated SDK provides Java and Python libraries, API bindings, utilities, and sample code to automate and integrate with VMware Cloud Foundation.

Why It’s Gated

  • Contains proprietary VMware/Broadcom code.
  • Ensures only licensed customers/partners can use it.
  • Helps control distribution of sensitive API bindings and internal tooling.

Why Gated SDKs Exist in VCF

VCF is not a generic API platform; it is a validated system.
Some APIs:

  • Control core lifecycle
  • Mutate global topology
  • Bypass normal UI guardrails
  • Can easily create unsupported states

Exposing these APIs broadly would:

  • Fragment platform behavior
  • Break upgrade guarantees
  • Explode support matrices

So VMware/Broadcom chooses controlled exposure instead of full openness.

What Typically Gets Gated

Gated SDKs often include APIs that:

  • Drive SDDC Manager internal workflows
  • Orchestrate LCM sequencing
  • Mutate domain-level state machines
  • Access pre-GA or evolving models
  • Bypass tenant-safe abstractions

These are platform-builder APIs, not tenant or end-user APIs.

What “Gated” Means Practically

A Gated SDK usually implies one or more of the following:

  • Explicit enablement flag
  • Contractual agreement
  • Limited documentation access
  • No backward-compatibility guarantees
  • Direct escalation path to engineering
  • Strong expectations of platform literacy

You are expected to know what you are doing.

Relationship to Public APIs

API Type Audience Stability Access
Public VCF APIs Customers High Open
VCF Automation APIs Tenants / DevOps High Open
Gated SDKs Platform builders Medium Restricted
Internal-only APIs VMware only Low Closed

A Gated SDK is official, but not democratized.

What a Gated SDK Is Not

It is not:

  • Unsupported
  • A hack
  • A private backdoor
  • A “use at your own risk” API

It is supported; but with conditions.

Architecturally Speaking…

If you see something described as a Gated SDK, it usually signals:

  • “This is powerful”
  • “This can break invariants”
  • “This is for platform-level integration”
  • “You need approval and discipline”

In other words:

This is not a convenience API — it is a responsibility API.

Summary

  • A Gated SDK is an official but restricted VCF SDK
  • It protects lifecycle safety and supportability
  • It is intended for platform builders, not tenants
  • Access implies higher expectations and accountability
  • It exists to balance power vs stability
