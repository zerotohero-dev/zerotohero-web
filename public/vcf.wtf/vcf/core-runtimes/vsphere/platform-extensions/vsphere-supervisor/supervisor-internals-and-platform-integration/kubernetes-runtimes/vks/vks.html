VKS

VKS is essentially the evolution of what was previously known as Tanzu Kubernetes Grid Service (TKGs).

“What was formerly known as Tanzu Kubernetes Grid Service (TKGs) is now unified under a single, intuitive name: vSphere Kubernetes Service (VKS).”

VKS is: Upstream Kubernetes + VMware’s opinionated lifecycle, security, and infrastructure integration.

Key Properties of VKS

  • Runs on vSphere
  • Worker nodes are VMs
  • Created and managed via vSphere Supervisor
  • Tightly integrated with:
    • vSphere networking (NSX / VDS)
    • vSphere storage (vSAN, CNS)
    • vSphere IAM / RBAC boundaries

Inside the Guest Cluster:

  • Kubernetes sees:
    • Namespaces
    • RBAC
    • CRDs
    • Workloads
  • It does not see:
    • Organizations
    • Projects
    • Approval flows
    • Business tenancy

Those live outside the cluster, enforced by:

  • vSphere
  • vSphere Supervisor
  • Automation layers (i.e., VCF Automation)

This is similar to saying “a Docker container does not know which Jira project paid for the VM it runs on”.

vSphere / ESX
└── Supervisor Cluster
    └── Supervisor Namespace
        └── TKG Workload Cluster (guest cluster)
            ├── Kubernetes API
            ├── Nodes = VMs
            └── Apps / Pods

VKS Components

VKS exposes three layers of controllers to manage the lifecycle of a VKS cluster.

1. VKS Core Components

VKS provisions clusters that include the components necessary to integrate with the underlying vSphere Namespace resources. These components include:

  • A Cloud Provider Plug-in that integrates with the Supervisor
  • Integration with VMware Cloud Native Storage (CNS) for persistent volumes

Persistent volume requests from a VKS cluster are passed to the Supervisor, which handles storage provisioning.

2. Cluster API (CAPI)

The Cluster API provides declarative, Kubernetes-style APIs for:

  • Creating clusters
  • Configuring clusters
  • Managing cluster lifecycle

Inputs to the Cluster API include:

  • A resource describing the cluster itself
  • Resources describing the virtual machines that make up the cluster
  • Resources describing cluster add-ons

3. Virtual Machine Service

The Virtual Machine Service provides a declarative, Kubernetes-style API for managing virtual machines and associated vSphere resources.

Key concepts include:

  • Virtual Machine Classes, which define reusable hardware configurations
  • Lifecycle management of control plane and worker node VMs that host a VKS cluster

VKS Cluster Components

The components that run inside a VKS cluster fall into four functional areas:

1. Authentication and Authorization

  • Authentication Webhook
    A webhook running inside the cluster that validates user authentication tokens.

2. Storage Integration

  • Container Storage Interface (CSI) Plugin
    A paravirtualized CSI driver that integrates with CNS through the Supervisor.

3. Networking

  • Container Network Interface (CNI) Plugin
    Provides pod networking capabilities.

4. Load Balancing

  • Cloud Provider Implementation
    Supports creation and management of Kubernetes LoadBalancer services.

VKS API

The VKS API is used to provision and manage VKS clusters. It is a declarative API accessed using kubectl and YAML manifests.

You can download the VMware-enhanced kubectl binary from the Supervisor API endpoint.

With the declarative model:

  • You define the desired state (number of nodes, VM sizes, Kubernetes version, storage, etc.)
  • VKS reconciles the actual state to match the desired state

To manage a cluster:

  1. Define the cluster configuration in YAML.
  2. Apply it using kubectl.
  3. Update the YAML to modify the cluster over time.
