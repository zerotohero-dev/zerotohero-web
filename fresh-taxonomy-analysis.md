## Recommended Tag Application Strategy

### Multi-Tag Strategy:
Each article should typically have **3-5 tags**:
1. **Primary Topic Tag** (e.g., "go", "kubernetes", "security")
2. **Subtopic Tag** (e.g., "error-handling", "deployment", "encryption")
3. **Content Type Tag** (e.g., "tutorial", "reference", "opinion")
4. **Difficulty Tag** (optional: "beginner", "intermediate", "advanced")
5. **Series Tag** (optional: if part of a series)

### Example Tag Assignments:

**File: `/content/inbox/docker-go.md` - "Building a Secure Webhook Server with Go and Docker"**
- Primary: `go`
- Secondary: `docker`, `security`, `webhooks`
- Type: `tutorial`
- Suggested Tags: `["go", "docker", "security", "http", "tutorial"]`

**File: `/content/spire/mtls/intro.md` - "Introduction to mTLS with SPIRE"**
- Primary: `spire`
- Secondary: `mtls`, `spiffe`, `security`
- Type: `tutorial`
- Suggested Tags: `["spire", "spiffe", "mtls", "security", "kubernetes", "tutorial"]`

**File: `/content/highlights/issue-0006.md` - "Earn the Promotion You Deserve"**
- Primary: `career`
