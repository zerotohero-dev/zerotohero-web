VM NIC (Virtual Machine Network Interface Card) in VMware virtualization

“vmnic-net-diagram 1.png” could not be found.

A VM NIC (often written vNIC) is a virtualized network adapter presented to a virtual machine by the hypervisor. From the guest OS’s point of view, it behaves like a physical Ethernet card; from the host’s point of view, it’s a software object that plugs into VMware’s virtual networking stack.

Effectively, it is a software Ethernet card that lives inside the VM.

At the VM layer

  • Appears as a standard network adapter (e.g., Intel or VMware-branded)
  • Has:
    • A MAC address
    • Link state (up/down)
    • One or more IP addresses assigned by the guest OS
  • Drivers inside the VM talk to it just like real hardware

At the Hypervisor Layer

  • The vNIC is backed by VMware virtual networking
  • It connects to:
    • A vSwitch (Standard or Distributed)
    • A Port Group
  • Traffic is then forwarded to:
    • Other VMs
    • The host
    • The physical network via uplinks (pNICs)

Common VM NIC types in VMware

VMware supports multiple vNIC “models,” each with different tradeoffs:

VM NIC Type Description When to use
E1000 / E1000e Emulates Intel 1 Gb NIC Legacy OS compatibility
VMXNET3 Paravirtualized, VMware-native Best default choice
Flexible Adapts based on guest OS Rarely used today

How a VM NIC connects to the Network

A typical path looks like this:

[ VM ]
  |
[vNIC]
  |
[Port Group]
  |
[vSwitch (VSS or VDS)]
  |
[pNIC / Uplink]
  |
[Physical Network]
  • Port Group defines policies (VLAN, security, shaping)
  • vSwitch is the virtual L2 switch
  • pNICs are the host’s physical NICs

VM NIC vs Physical NIC (pNIC)

VM NIC (vNIC) Physical NIC (pNIC)
Software-defined Hardware device
Lives inside a VM Lives on the ESXi host
Can be added/removed hot Requires hardware
Shares bandwidth Provides real bandwidth

Advanced VM NIC features

  • Multiple vNICs per VM
    • Separate traffic (management, data, backup)
  • SR-IOV
    • VM bypasses vSwitch and talks almost directly to hardware
    • Very high performance, less flexibility
  • Traffic Shaping and Security
    • Promiscuous mode
    • MAC changes
    • Forged transmits

Summary

  • A VM NIC (vNIC) is a virtual Ethernet adapter attached to a VM
  • It connects the VM to VMware’s virtual networking stack
  • VMXNET3 is the modern, high-performance choice
  • Networking is enforced through port groups and vSwitches
  • Multiple vNICs, policies, and offloads make it far more flexible than physical NICs
