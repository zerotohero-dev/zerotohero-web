What is a VKS Cluster

A 

VKS

 cluster is a Kubernetes cluster that is built and supported by VMware and that runs your application workloads. 

VKS

 clusters are part of a 

vSphere Kubernetes Service

 ( 

VKS

 ) deployment. You use 

VKS

 cluster management to manage 

VKS

 clusters. Their life cycle, however, is handled by the native 

Supervisor

 APIs.

Relation Between 

VKS

 and 

VCF Automation

VKS

 provides self-service life cycle management of Kubernetes workload clusters. 

VKS

 is an implementation of the open-source Cluster API project that defines a set of custom resources and controllers to manage the life cycle of Kubernetes clusters. 

VKS

 is a component of 

Supervisor

. The 

Supervisor

 is a source of compute resources that can be registered to 

VCF Automation

. System administrators allocate the 

Supervisor

 resources to the organizations in 

VCF Automation

. The 

Supervisor

 itself does not belong to any organization. Project acministrators request 

vSphere Namespaces

 for projects, and 

VCF Automation

 allocates 

vSphere Namespaces

 matching the request.

A single 

Supervisor

 can provide resources for multiple 

VCF Automation

 organizations.

Within a single organization, you can move 

vSphere Namespaces

 from one project to another.

For more information about organizations and projects within the context of 

VKS

 cluster management, see Hierarchy of the Resources That VKS Cluster Management Works With.

For more information about 

VKS

, see vSphere Supervisor Platform and the 

Managing vSphere Kubernetes Service with vSphere Supervisor

 documentation.

Agents and Extensions Installed on the 

VKS

 Cluster

To function properly, 

VKS

 cluster management depends on a set of agents and extensions that run in all 

VKS

 clusters. When a 

VKS

 cluster is attached to 

VKS

 cluster management, the 

Supervisor

 auto-attach service installs a set of extensions and cluster resource definitions to the cluster nodes.

The agents configuration contains information about the correct 

VKS

 cluster management endpoint, the image registry that provides the images, and the proxy configuration.

Cluster Health

The overall health of a cluster is an aggregation of the health of the components and nodes in the cluster.

The health status of the cluster can be one of the following values:

  • Healthy

    A cluster is healthy when all nodes and components are healthy, and a heartbeat for the cluster is received every minute.

  • Unhealthy

    A cluster is unhealthy if either of the following is reported as unhealthy:

    • One or more of the cluster’s control plane nodes

    • One or more of the cluster’s components

  • Warning

    • A cluster can have a warning status if any of its worker nodes are in an unhealthy or unknown state.

    • A cluster can also have a warning status if any nodes (worker or control plane) are in a warning state.

  • Unknown

    The health status of a cluster in unknown if either of the following is reported as unknown:

    • One or more of the cluster’s control plane nodes

    • One or more of the cluster’s components

  • Disconnected

    A cluster is considered disconnected if no heartbeat is received from the cluster for more than 3 minutes.

Node Health

The information received for a node consists of the 

NodeReady

 condition, and a number of other conditions like 

MemoryPressure

, 

DiskPressure

, and 

OutOfDisk

. 

VKS

 cluster management uses the value of these conditions to discover the health of the node. The status of each condition is assumed to be unchanged until the node reports a change.

Based on the reported conditions, the health status of a node can be one of the following:

  • Healthy

    If 

    NodeReady

     is 

    True

     and all other conditions are healthy, the node is healthy.

  • Unhealthy

    The node is unhealthy if 

    NodeReady

     is 

    False

    . The node is also unhealthy if 

    NodeReady

     is 

    True

    , but more than half of the other conditions are in an unhealthy state.

  • Warning

    The warning status indicates that 

    NodeReady

     is 

    True

    , and less than half of the other conditions are in an unhealthy state.

  • Unknown

    If 

    NodeReady

     has any value other than 

    True

     or 

    False

    , the health status of the node is 

    Unknown

    . The node can also have an unknown status if 

    VKS

     cluster management receives no heartbeat from the cluster for more than three minutes.

Component Health

VKS

 cluster management monitors the health of the following components running in the cluster:

  • The 

    kube-apiserver

  • The 

    scheduler

  • The 

    controller-manager

  • One or more 

    etcd

     components (

    etcd-0

    , 

    etcd-1

    , 

    etcd-2

    , and so on)

The component status event reports the 

Healthy

 condition for each of these components every 45 seconds.

The health status of each component can be one of the following:

  • Healthy

    If the last reported value of the 

    Healthy

     condition of the component is 

    True

    , the component is 

    Healthy

    .

  • Unhealthy

    If the last reported value of the 

    Healthy

     condition of the component is 

    False

    , the component is 

    Unhealthy

    .

  • Unknown

    If the last reported value of the 

    Healthy

     condition of the component is 

    Unknown

     or it is something other than 

    True

     or 

    False

    , the health status of the component is 

    Unknown

    . The component can also be in this state if 

    VKS

     cluster management receives no heartbeat from the cluster for more than three minutes
