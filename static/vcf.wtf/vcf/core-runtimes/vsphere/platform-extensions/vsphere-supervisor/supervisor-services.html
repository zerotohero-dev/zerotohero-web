Supervisor Services

Supervisor Services are platform-level extensions for the vSphere Supervisor that provide cloud-native infrastructure capabilities by packaging functionality as Kubernetes Operators and service bundles that run on the Supervisor Cluster.

These services enable foundational capabilities such as Kubernetes consumption, storage services, registry operations, DNS management, ingress controllers, and more— all integrated directly into the vSphere control plane.

Supervisor Services are:

  • Installed and managed per Supervisor Cluster
  • Controlled by platform administrators
  • Versioned and upgradeable independently
  • Consumed by tenants or platform components

Core Supervisor Services

Service Name Purpose
vSphere Kubernetes Service (VKS) Enables provisioning and lifecycle management of Kubernetes clusters via the Supervisor
Local Consumption Interface UI/UX extension for Namespace resource consumption within the vSphere Client
vSAN Data Persistence Platform (vDPP) Services Provides stateful storage (e.g., object storage via MinIO) integrated with vSAN
Backup and Recovery Service–Velero Kubernetes backup/restore and disaster recovery for workloads
Certificate Management Service–CA Cluster Issuer Issuer service to manage certificates for Kubernetes workloads
Cloud Native Registry Service– Harbor Container image registry service with image signing, scanning, and replication
Kubernetes Ingress Controller Service– Contour Ingress controller to manage external traffic into Kubernetes workloads
External DNS Service –ExternalDNS Dynamic DNS record management for workloads
Supervisor Management Proxy Provides connectivity between management and workload networks where isolation exists
NSX Management Proxy (deprecated) Older proxy for NSX connectivity; replaced by Supervisor Management Proxy
Data Services Manager Consumption Operator Enables consumption of data services via Supervisor APIs
Secret Store Service Native secret store service for Kubernetes workloads
ArgoCD Service GitOps continuous delivery service for declarative app deployment

Labs / Experimental Supervisor Services

In addition to core production Supervisor Services, there is a Supervisor Services Labs Catalog that includes experimental or community services which can be installed on Supervisor Clusters but are not yet part of the core catalog.

Labs Services Include

Service Use Case
External Secrets Operator Bridge between external secret stores and Kubernetes workloads
RabbitMQ Cluster Kubernetes Operator Messaging broker operator for RabbitMQ clusters
Redis Operator Redis cache management operator
KEDA Kubernetes Event-Driven Autoscaler

Labs services are typically not supported at the same guarantee level as core Supervisor Services and may be subject to change.

Detailed Service Descriptions

vSphere Kubernetes Service (VKS)

  • Formerly known as the Tanzu Kubernetes Grid Service
  • Provides APIs to create and manage Kubernetes clusters
  • Enables declarative Kubernetes lifecycle management via Supervisor APIs.

Local Consumption Interface

  • UI component for resource consumption inside Namespaces
  • Displays resources in the vSphere Client UI under Workload Management
  • Improves namespace Kubernetes consumption UX

vSAN Data Persistence Platform (vDPP)

  • Framework for stateful services using vSAN
  • Enables highly available object and block storage services for cloud-native workloads

Backup and Recovery Service—Velero

  • Integrates Velero with Supervisor Kubernetes
  • Enables backup/restore of Kubernetes resources and volumes
  • Natively integrated into Supervisor cluster workflows

Certificate Management Service—CA Cluster Issuer

  • Leverages cert-manager primitives
  • Issues certificates for workloads running on Supervisor or tenant clusters
  • Helps with workload TLS termination and secure communication

Harbor Registry Service

  • Adds a container registry within the Supervisor cluster
  • Supports image scanning, signing, and lifecycle policies
  • Works with ingress and DNS services for external access

Ingress and DNS Services

Ingress Controller (Contour):

  • Provides ingress routing and load balancing for Kubernetes workloads
  • Integrates with Envoy for traffic proxying
    ExternalDNS:
  • Automatically updates DNS records based on exposed service endpoints
  • Connects to external DNS providers for dynamic record management

Proxy Services

Supervisor Management Proxy:

  • Facilitates communication between management and workload networks when isolated
  • Ensures Supervisor components can reach required management endpoints
    NSX Management Proxy:
  • Previously used for similar connectivity needs but is now deprecated in favor of Supervisor Management Proxy

Data Services Manager Consumption Operator

  • Enables native Supervisor APIs to expose data services to tenants
  • Integrates the Data Services Manager (DSM) with Supervisor Cluster workload consumption paths

Secret Store and ArgoCD

Secret Store Service:

  • Native secret storage capabilities for Kubernetes workloads
    ArgoCD Service:
  • Provides a GitOps delivery model for application deployments
  • Useful for declarative automation and continuous deployment workflows

Summary

  • Supervisor Services are the core platform building blocks that make vSphere Supervisor extensible and cloud-capable.
  • These services cover Kubernetes lifecycle, storage, registry, ingress, DNS, certificate management, backup, data services consumption, secrets, and more.
  • They are installed per Supervisor Cluster, managed by platform administrators, and consumed by tenant workloads or automation layers like VCF Automation.
  • There is also a Labs Catalog for experimental or early-stage Supervisor Services.
