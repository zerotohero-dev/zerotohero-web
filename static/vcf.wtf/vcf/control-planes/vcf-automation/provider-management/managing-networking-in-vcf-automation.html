Managing Networking in VCF Automation

As an organization administrator, you can view what networking infrastructure is assigned to your organization by the provider administrator and you can manage networking infrastructure within the organization.

Virtual Private Clouds (VPCs)

A VPC provides an isolated domain of networking resources that can be associated with one or more namespaces. You can create VPCs for your organization and associate each VPC to one or more namespaces.

Each VPC can leverage 3 different types of IP blocks that each have different levels of reachability:

  • Private CIDRs are local to the VPC and require NAT to connect to any resources outside of the VPC. These CIDRs are managed by the VPC administrator and do not need to be unique across multiple VPCs.

  • Transit Gateway (TGW) Private IP blocks are only advertised within the 

    VCF Automation

     organization and its TGW. The available IP blocks are defined by the organization administrator. Project administrators can allocate subnets from within the defined IP Blocks for use in their VPCs. These IPs have direct reachability to other VPCs within the organization using the TGW Private IP space.

  • External IP Blocks are managed by the provider administrator. IPs in this space are used for outbound Source NAT to enable external reachability for workloads using Private IPs. Organization administrators can assign subnets within the external IP blocks by the allocations defined by the Provider.

You can create a new VPC for each namespace for workloads that require some degree of network separation. If no network separation is required, you can use the same VPC for multiple namespaces.

The Network Service can be used to create, update, manage, and delete resources within the VPC by organization administrators and advanced project users.

Connectivity profiles

A connectivity profile is a template that allows simplified configuration of new VPCs. A default connectivity profile is configured by the provider administrator to simplify the initial organization configuration. You can provide one or more external and private transit gateway IP address blocks that are used by VPCs configured with a specific connectivity profile.

Transit gateways

Depending on the regional quota, each organization has a transit gateway which provides connectivity to the provider gateway within the organization. One or more VPCs are connected to the transit gateway, and that connection is defined by a VPC connectivity profile. Each VPC has connected workloads and a private subnet. SNAT rules translate addresses from this private subnet to a public address in the IP spaces block. This infrastructure enables the organization and its workloads to connect to external networks.
