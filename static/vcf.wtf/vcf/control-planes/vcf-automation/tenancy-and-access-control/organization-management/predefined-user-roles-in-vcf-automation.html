Predefined User Roles in VCF Automation

Roles assigned to users and groups are an essential part to identity and access management in 

VCF Automation

. You can leverage identity providers with 

VCF Automation

 organizations to enable single sign-on to provision users and groups. Once the users and groups are added to the 

VCF Automation

 organization, they must be assigned roles to enable specific permissions and access to different features within 

VCF Automation

.

What are User Roles in 

VCF Automation

VCF Automation

 has several levels of user roles. These different levels control access to the organization, the projects that produce or consume the blueprints, catalog items, access to IaaS services, and the ability for users to use or see individual parts of the user interface. These different levels give organization administrators different tools to apply any level of granularity that is required by their operational needs.

As the organization administrator, after you configure an identity provider for your organization, you can import users and groups and manage their role assignments using the 

Access Control

 page. There are two types of predefined roles in 

VCF Automation

: organization roles and project roles. In addition, you can create custom roles for specific use cases.

Predefined Organization Roles

The organization roles are global. The predefined organization-level roles are Organization Administrator, Organization Auditor, and Organization User.

Organizational roles define permissions within the organization. Organization administrators can add and manage other users and create custom roles at the organization level. Custom roles allow you to tailor permissions to specific user requirements.

Role General permissions
Organization Administrator Can access the organization portal and add users and groups to the organization.

Organization administrators can:

- Manage projects and project users

- Configure integrations, such as Git

- Manage blueprints and catalog items

- Manage namespaces and associated resources

- Fully access namespaces that they did not create

- Manage new VPCs and associated resources

- See all projects and all deployments

- Create and manage content libraries

- Create and manage policies on resources and deployments

- Create and manage event subscriptions
Organization User Can log in to the organization portal. Must be added to a project.
Organization Auditor Read-only version of the organization administrator.
Defer to Identity Provider User permissions are determined based on information received from the identity provider for the organization.

Predefined organization-level roles are managed by the provider administrator in the 

VCF Automation

 Provider Management Portal. For a full list of what rights are given to each role, see Predefined VCF Automation Roles and Their Rights in the Provider Management documentation.

Custom Organization Roles

As an organization administrator, you can create custom roles for your organization.

The custom roles have two categories, view and manage.

  • View. A user assigned to a role with this permission can see all the items for all projects across the organization in the selected sections of the user interface. This role is useful for users who need to see accounts, configurations, or assigned values.

  • Manage. A user assigned to a role with this permission can see all the items and has full add, edit, and delete permissions for all projects across the organization in the selected sections of the user interface.

These permissions extend the privileges that are granted by the other roles and are not restricted by project membership. For example, you can expand a project administrator's permissions to manage parts of the infrastructure or give an organization auditor an ability to review and respond to approvals requests.

For instructions on how to create a custom role, see Create a Custom Role in for All Apps.

Project Roles

In addition to the organizational roles, 

VCF Automation

 has project roles. Project members can view and manage project resources depending on their project role. Project roles include Project Administrator, Project Advanced User, Project User, and Project Auditor.

These roles are assigned automatically on a per project basis in 

VCF Automation

. The project roles can vary between projects. The same user can be an administrator in one project and an advanced user in another project.

Role General permissions
Project Administrator Project administrators leverage the infrastructure that is created by the organization administrator to ensure that their project members have the resources they need for their development work.

Project administrators can:

- View all content for their project, including namespaces, deployments, and so on

- Perform all Day 2 actions on resources

- Manage users in the project and assign roles

- Manage project-level settings and administrator content

- Create resources using the IaaS services console in namespaces they created or have access to

- Manage catalog approval requests
Project Advanced User Advanced users work within their projects to request catalog items and create resources using the services console. They can perform day 2 actions on all resources.
Project User Project users can only access the catalog and request catalog items. They can perform day 2 actions on their own deployments. They cannot use the IaaS services console.
Project Auditor Project auditors are restricted to read-only access.
