vCenter Roles for Supervisor Namespaces

The VCF 9.0 release introduces a new role called 

Supervisor Administrator

. The 

Supervisor Administrator

 role is granted by default to members of Administrators group on the 

Namespaces

 resource pool and folder in newly deployed 

Supervisors

 instead of the Read-only role in the prior versions.

Based on the 

vCenter

 role assigned within the 

Supervisor

 inventory, a user or group can be assigned one of the following roles:

Administrator Role

As a vSphere Administrator, you can view and manage the vSphere infrastructure, including 

VKS

 clusters, 

vSphere Pods

 and VM service VMs.

The vSphere Administrator has the membership of the "Administrators SSO Group".

This Administrators group is automatically assigned the 

Supervisor Administrator

 role on the 

Supervisor

 hierarchy.

Supervisor

 Administrator Role

A 

Supervisor

 Administrator

 role can only be assigned on the 

Supervisor

 folder.

You will be able to view the namespaces inventory only after assigning it the 

Read-only

 role on the Cluster.

When the 

Supervisor

 Administrator

 role is assigned on the 

Supervisor

 folder, a context is created that givess access only to cluster level resources in the 

Supervisor

 and not the 

vSphere Namespace

. That is, you cannot list or view 

vSphere Namespaces

. To be able to view 

vSphere Namespaces

 when you log in through VCF CLI, you must have permissions on the specific 

vSphere Namespace

.

Namespace Owner/Edit/View Role

In this role, you are a consumer of the 

vSphere Namespace

 and can provision 

VKS

 clusters and vSphere Pods using kubectl.

Given the Supervisor cluster IP address, you can login and start deploying and managing workloads on the 

vSphere Namespace

 you have been granted access to.

vCenter

Alarms.SetStatus		 Supervisor

 folder		 As a user with 

Alarms

 privileges on the 

Supervisor

 folder you can perform the following actions on the inventory.

- Acknowledge Alarm: Suppresses alarm actions on triggered alarms.


- Create Alarm: Allows creating new alarm definitions.


- Modify Alarm: Allows changing the settings of an existing alarm.


- Disable Alarm Action: Prevents alarm actions from occurring after an alarm is triggered, but the alarm remains active.


- Disable or Enable Alarm on Entity: Allows toggling the activation of a specific alarm on a target object.


- Set Alarm Status: Allows changing the status of an alarm, for example, from triggered to cleared.

VirtualMachine.GuestOperations.Execute		 Supervisor

 folder		 As a user with 

GuestOperations

 privileges, you can perform the following actions on the inventory:

- Guest Operation Queries: Allows virtual machine guest operations that involve querying the guest operating system, such as listing files in the guest operating system.


- Guest Operation Alias query: Allows virtual machine guest operations that involve querying the alias for the virtual machine.

- Guest Operation Alias modifications: Allows virtual machine guest operations that involve modifying the alias for the virtual machine.

- Guest Operation Modifications: Allows virtual machine guest operations that involve modifications to a guest operating system in a virtual machine, such as transferring a file to the virtual machine.

- Guest Operation Program Execution: Allows virtual machine guest operations that involve running an application in the virtual machine.


With these set of privileges user can perform troubleshooting or support bundle collection when network is not available.
