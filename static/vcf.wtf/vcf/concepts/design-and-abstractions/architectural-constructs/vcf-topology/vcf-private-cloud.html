VCF Private Cloud

#tenancy

A VCF Private Cloud is the highest level of management and consumption for the underlying SDDC resources. A VCF Private Cloud can contain one or more VCF Fleets.

VCF Private Cloud is the customer-operated private cloud deployment model built on VMware Cloud Foundation.
It delivers cloud-like operations, Governance, and lifecycle management on customer-owned or customer-controlled infrastructure (on-prem or dedicated colocation).

VCF Private Cloud = “You run the cloud, using VMware’s cloud platform.”

VCF Private Cloud is opinionated and integrated:

  • Single lifecycle authority (SDDC Manager)
  • Validated stack (ESX, vCenter, NSX, vSAN)
  • Declarative operations
  • Strong separation of platform vs tenant
  • Built-in multi-tenancy
  • Day-0 → Day-2 automation

This is fundamentally different from “installing vSphere and calling it a cloud.”

Core Components

Layer Role
SDDC Manager Platform lifecycle & topology
vCenter Server Compute & cluster management
NSX Networking & security
vSAN Software-defined storage
VCF Automation Consumption, policy, observability

These components are installed, upgraded, and validated as one system.

Control Plane vs Consumption Plane

VCF Private Cloud is deliberately split:

Platform Control Plane

  • Runs in the Management Domain / Management Cluster
  • Controls lifecycle, upgrades, topology

Consumption Plane

  • Used by Tenants
  • Exposed via Organizations, Projects, policies
  • Runs workloads in Workload (VI) Domains / Clusters

This mirrors public cloud responsibility boundaries.

Multi-Tenancy Model

VCF Private Cloud supports enterprise-grade multi-tenancy:

  • Organizations → business tenants

  • Projects → sub-tenants (teams, environments)

  • Namespaces / Resource Pools → execution boundaries

  • NSX Segmentation → network isolation

  • Domains / clusters → failure & lifecycle isolation

Tenancy is logical and policy-driven, not per-cluster by default.

Where Does VCF Private Cloud Run

VCF Private Cloud can be deployed on:

  • On-premises data centers
  • Dedicated colocation
  • Partner-hosted environments

What matters is control, not location. VCF optimizes for control and predictability, not infinite elasticity:

  • VCF Private Cloud ≈ “like AWS, but you run AWS yourself”
  • SDDC Manager ≈ Cloud provider internal control plane
  • Organizations / Projects ≈ Accounts and subscriptions
  • Workload Clusters ≈ Capacity pools

Summary

  • VCF Private Cloud is customer-operated cloud infrastructure
  • Built on VMware Cloud Foundation
  • Strong separation of platform vs tenant
  • Enterprise-grade Multi-Tenancy and lifecycle
  • Cloud outcomes without giving up control
